## 二、前端 API 请求封装修复：POST body 丢失导致后端收到空请求体

### 现象
- `POST /api/repos/{id}/tasks` 后端报 `title required`，同时后端日志显示 `req.body` 为空。
- 前端页面显示“将发送的 JSON”正常，但实际网络请求未带 body。

### 根因
- `frontend/src/api/client.js` 的 `apiPost()` 未正确传递 `fetch` 的 `body` 或未设置 `Content-Type: application/json`，导致后端收到空 body，解析后变 `{}` → `items[0]={}` → `title required`。

### 修复
- 修复 `apiPost()`：支持 `body` 参数，自动 `JSON.stringify`，并默认设置 `Content-Type: application/json`。
- 新增 `apiDelete()`：用于仓库删除等操作（若已实现）。

## 三、后端 tasks 入库错误可诊断性增强

- `POST /api/repos/{id}/tasks` 在校验失败时返回：
  - `error`：具体错误（如 `title required`）
  - `index`：失败元素在 items 数组中的下标
  - `bad_item`：触发错误的原始 JSON 元素
- 在后端日志中打印关键调试信息（可选）：`Content-Type`、`Content-Length`、`req.body.size()`、`body_head`，便于定位请求体丢失/编码错误。

## 四、同步（sync）后自动生成任务的健壮性改进

- 仓库页面同步成功后自动触发任务生成时：
  - AI 输出不为纯 JSON 时，通过提取策略解析（`extractJsonObject`）。
  - 解析失败时展示 AI 原始输出（调试字段），并将错误限定在“任务生成失败”，不影响同步本身的成功结果展示。

- 修复了snapshots 和 issues, pulls 解析时可能不为object导致错误的Bug
