# 今日修改日志（2026-04-25）

## 一、新增：仓库介绍（Repo Introduction）自动生成与展示

- 目标：每个仓库在**首次同步成功后**自动生成一段**纯文本**介绍，说明仓库做什么、有什么用，并存入数据库，在详情页展示。
- 生成方式：复用后端 AI（RAG/ask）能力生成介绍文本。
- 更新策略：**仅首次同步生成一次**；后续同步不重复生成，降低耗时与成本。
- 展示位置：仅在 `RepoDetailView` 展示“仓库介绍”卡片（只读，展示最新介绍与更新时间）。

## 二、数据库变更：repos 表新增介绍字段（迁移）

- 在 `repos` 表新增字段：
  - `intro_text`：介绍正文（TEXT，默认空字符串）
  - `intro_updated_at`：更新时间（TEXT，datetime('now')）
- 迁移策略：在 `Db::init_schema()` 中执行 `ALTER TABLE ... ADD COLUMN`，兼容已有数据库文件。

## 三、后端：同步后落库逻辑与修复

- 在 `POST /api/repos/{id}/sync` 同步流程末尾增加：generate_and_store_repo_intro
  - 若 `repos.intro_text` 为空，则调用 AI 生成介绍
  - 调用 `UPDATE repos SET intro_text=?, intro_updated_at=datetime('now') WHERE id=?;` 写入数据库
- 修复：

- 新增读取接口：
  - `GET /api/repos/{id}/intro`：返回 `{ ok, repo_id, intro_text, intro_updated_at }`

## 四、前端：RepoDetailView 拉取并显示 /intro

- `RepoDetailView.vue` 增加介绍展示卡片：
  - 页面加载时调用 `GET /api/repos/{id}/intro`
  - 修复字段映射：后端返回 `intro_text`，前端使用 `data.intro_text` 赋值到 `introText`
  - 无介绍时显示占位提示：“暂无介绍（首次同步成功后会自动生成）”

## 五、同步性能：禁用同步后自动生成任务（避免测试过慢）

- 在 `ReposView.vue` 中将“同步后自动生成任务清单并跳转”功能禁用/注释，避免测试场景下触发 AI 任务生成导致同步耗时过长。

## 六、新增：同步接口参数透传（全量/增量）

- 在 `ReposView.vue` 增加同步参数 UI 与请求透传：
  - `mode=full`：传递 `issues/pulls/commits/releases` 的 `page_start + pages_count`，以及 `ci_pages_count`
  - `mode=incremental`：传递各项 `*_pages_count` 作为拉取上限（后端会按 cursor 增量推进）
- 使用 `URLSearchParams` 组装 query，避免手拼参数出错。
